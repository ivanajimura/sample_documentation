src/app Documentation

This document pertains to the src/app folder, which contains the modules responsible for effectively generating the Data Package itself, and generating the Software Configuration Index docx document.

Are you lost... maybe?

This document is part of a series of documents to help onboard new developers to the software Certifiable Documentation Generator (hereby referred to as CDG). This is not a good starting point to understand CDG, so if you have not yet read the following documents, you are highly recommended to do so before continuing reading this one.

Modules

dataPackage.py

This module is responsible for generating the data package based on a list of Jira issues. This is likely one of the most complexes modules in the codebase -- and has the highest Cognitive Complexity value according to SonarQube reports (100, in fact) --, so it may take some time and code runs to grasp all this module does. Most of the code is legacy to the authors of this document, and some of it has become unnecessary due to refactors. The new developers are recommended to refactor this module to reduce cognitive complexity and improve performance.

SCI.py

This module is responsible for creating a docx document called Software Configuration Index (SCI), which can be used by the user to identify potential problems during the certification and to provide relevant data for the certification body. Although SonarQube does not flag this module as the most complex in the codebase, it can be quite challenging to comprehend due to the docx library heavily used here. This module was legacy to the authors of this document, and only minor changes were made to it in order to add compatibility with Linux and Mac OSes. There is a chance that docx documents are no longer compatible with IBM DOORS Next Gen, so we decided against refactoring this module until we were more certain, as there would be a risk of refactoring a module that would no longer be useful anyway. As of the end of the tenure of the MSE 23-24 cohort, this was still unclear, so we recommend the new developers to ensure the viability of docx before refactoring this module.

CommonValues.py

This module is heavily used by SCI in order to split artifacts into their correct development stage. In order to reduce complexity, this could be moved into the Config folder into the data_packages_stages.json file. This module was not edited by the 23-24 cohort.
