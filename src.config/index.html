src/app Documentation

This document pertains to the src/config folder, which contains the json files used for congifuring the software.

Are you lost... maybe?

This document is part of a series of documents to help onboard new developers to the software Certifiable Documentation Generator (hereby referred to as CDG). This is not a good starting point to understand CDG, so if you have not yet read the following documents, you are highly recommended to do so before continuing reading this one.

End-of-Course Presentation of MSE 23-24 cohort CDG Installation and User Manual CDG Architecture Jira Set-up Main Readme.MD of this codebase

Modules

data_package_config.json

This file can be ignored as it has been deprecated. In v1.0, the settings were fixed in this file and any changes requested by the user required either the user to be able to edit this file and re-generate the binary, or a developer to make the changes, generate the binary, then send it back to the user. This file was kept as we were unable to ascertain it is never read by any other module, and, thus, removing it would not produce errors.

data_package_stages.json

This file contains all the development stages and naming patterns associated to them. In CDG Installation and User Manual, there is a section dedicated to the usage of these patterns.

environment.example.json

This file is deprecated. In v1.0, the user settings were saved into and read from this file. These are now inserted directly by the user from within the software, and the shelve library is used to store this data. This file was kept as we were unable to ascertain it is never read by any other module, and, thus, removing it would not produce errors.
